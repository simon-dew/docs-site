Troubleshooting and FAQs

This section lists most common errors and gives general tips.

Generating Diagnostics Information

To generate diagnostics information you can use one of the following ways:

Using Web Console

To generate diagnostics information using the Web Console, click on the Logs section from the menu on the left then select Collect Information as shown below.

You may choose to generate diagnostics information for all nodes or specific ones. After that, click on Start Collection. When the collection is done, the path in which the diagnostics information was stored will be displayed for each selected node.

The diagnostic information for every node running the Analytics Service should contain the following files:

File Log Content

ns_server.analytics_info.log

Troubleshooting log for the Analytics Service.

analytics_diag.json

Analytics Service node configuration and thread dumps.

Using CLI Command

You may also use the CLI command cbcollect_info to generate the diagnostics information.

Reporting an Issue

Good bug reports save a lot of time for everyone involved. During the developer preview, we suggest that you post your questions and bug reports to the Couchbase Forum. When reporting an issue, please include the following information:

  • Description of the issue

  • Diagnostics information as described in Generating Diagnostics Information section

  • Steps to reproduce the issue

Sending Feedback

We’d love to hear from you! Feel free to post your feedback to the Couchbase Forum.

FAQs

Q: What’s the relationship between N1QL for Query, N1QL for Analytics, and SQL++?

SQL++ is a formal language specification with well-defined semantics that was designed with the goal of providing a clean, unifying, powerful, declarative query language for handling semi-structured data. N1QL for Analytics is a Couchbase implementation of SQL++ and is the language used for Analytics. N1QL for Query is the query language used for the Couchbase Server query service. The itemized differences between N1QL for Query and N1QL for Analytics are summarized here.

Q: Can I use this with memcached buckets?

No, Analytics relies on DCP and JSON formatted documents. It can only be used with JSON coming from a Couchbase bucket.
