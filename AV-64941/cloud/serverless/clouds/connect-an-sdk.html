Connect Your SDK

    SDKs enable you to connect client code in popular languages to Capella.

    After you sign up for a Capella Database, get started with connecting an app.

    You can build something quickly using our Data API, or connect an app built with one of our ten Server SDKs.

    SDKs

    For more information on the SDKs and where to find their respective documentation, see Choose Your Language. If you are using Capella Serverless as part of the private preview, use the connection information in the next section.

    Connecting your SDK to Serverless

    The SDK Compatibility page lists the minimum supported SDK versions. Couchbase recommends using the latest version of your chosen SDK, as it contains bug fixes and feature enhancements, and makes working with Serverless much easier.

    The Getting Started page for each SDK features a Hello World program to show you how to connect to a Capella databse. The Getting Started pages also cover installing the SDK, and link to pages with further platform information. For links to each SDK, see Choose Your Language.

    These code samples work with Capella Serverless, with the following terminology differences:

    1. The client code asks for a bucket_name. This is the name you gave your database, with a random alphanumeric (Database ID) added — to protect against two databases being given the same name. You can find the database name in this form, in the Capella UI — go to Data API  General.

    2. The client code asks for a username and password.

      • For username, use the API Key.

      • For password, use the API Secret.

      • To generate API keys, go to Data API  API Keys, and Click Generate Key. A more detailed guide is given in the Generate API Keys section of the API keys page.

    import com.couchbase.client.java.*;
import com.couchbase.client.java.kv.*;
import com.couchbase.client.java.json.*;
import com.couchbase.client.java.query.*;

import java.time.Duration;

public class StartUsingCapella {

  // User input needed to replace the following <<values>>:
  static String connectionString = "couchbases://cb.<<your-bucket-name>>.cloud.couchbase.com";
  static String username = "<<your-API_Key>>";
  static String password = "<<your-API_Secret>>";
  static String bucketName = "<<your-bucket-name>>";

  public static void main(String... args) {
    Cluster cluster = Cluster.connect(
        connectionString,
        ClusterOptions.clusterOptions(username, password).environment(env -> {
          // Sets a pre-configured profile called "wan-development" to help avoid
          // latency issues when accessing Capella from a different Wide Area Network
          // or Availability Zone (e.g. your laptop).
          env.applyProfile("wan-development");
        })
    );

    // Get a bucket reference
    Bucket bucket = cluster.bucket(bucketName);
    bucket.waitUntilReady(Duration.ofSeconds(10));

    // Get a user-defined collection reference
    Scope scope = bucket.scope("samples");
    Collection collection = scope.collection("airline");

    // Upsert Document
    MutationResult upsertResult = collection.upsert(
        "my-document",
        JsonObject.create().put("name", "Xanadu Air")
    );

    // Get Document
    GetResult getResult = collection.get("my-document");
    String name = getResult.contentAsObject().getString("name");
    System.out.println(name); // name == "Xanadu Air"


    // Call the query() method on the scope object and store the result.
    QueryResult result = scope.query("SELECT * FROM airline WHERE id = 10;");

    // Return the result rows with the rowsAsObject() method and print to the terminal.
    System.out.println(result.rowsAsObject());
  }
}

// User input needed to replace the following <<values>>:
const DB_NAME = '<<your-bucket-name>>'
const ENDPOINT = `cb.${DB_NAME}.sdk.cloud.couchbase.com`
const USER = '<<your-API_Key>>'
const PW = '<<your-API_Secret>>'

async function connect() {
  try {
    const cluster = await couchbase.connect(`couchbases://${ENDPOINT}`, {
      username: USER,
      password: PW,
      configProfile: 'wanDevelopment',
    })
    const bucket = cluster.bucket(DB_NAME)
    const scope = bucket.scope('samples')
    const collection = scope.collection('airline')

    const res = await collection.get('airline_10')
    console.log(res.content)

    const qRes = await scope.query(`SELECT * FROM \`${collection.name}\` LIMIT 10;`)
    qRes.rows.forEach((r) => {
      console.log('Found row:', r)
    })

  } catch (e) {
    throw e;
  }
}
connect()

    console.log('Finished.  Exiting app...')
    process.exit(0)
  })
  .catch((err) => {
    console.log('ERR: ', err)
    console.log('Exiting app...')
    process.exit(0)
  });

    import sys

from couchbase.auth import PasswordAuthenticator
from couchbase.cluster import Cluster
from couchbase.options import ClusterOptions

# User input needed to replace the following <<values>>:
DB_NAME = '<<your-bucket-name>>'
ENDPOINT = f'cb.{DB_NAME}.sdk.cloud.couchbase.com'
USER = '<<your-API_Key>>'
PW = '<<your-API_Secret>>'

def main():
    try:
        auth = PasswordAuthenticator(USER, PW)
        options = ClusterOptions(auth)
        options.apply_profile('wan_development')
        cluster = Cluster.connect(f'couchbases://{ENDPOINT}', options)
        bucket = cluster.bucket(DB_NAME)
        scope = bucket.scope('samples')
        collection = scope.collection('airline')

        res = collection.get('airline_10')
        print(res.content_as[dict])

        q_res = scope.query(f'SELECT * FROM {collection.name} LIMIT 10')
        for r in q_res:
            print(f'Found row: {r}')

    except Exception as ex:
        print(f'Caught exception trying to connect Ex: {sys.exc_info()[0]}')


if __name__ == '__main__':
    main()
    Connection Exceptions

    Some platforms need a workaround to find or trust the certificate, as they don’t use the expected OpenSSL — currently Python on Windows, Node.js on Windows, and Node.js on MacOS.

    The simplest workaround is to set tls_verify=none in the connection string. For more information, see Certificates.

    Connection Troubleshooting

    The documentation for each SDK provides guidance for working around problems that may arise in development, when your development laptop is most likely not in the same cloud provider Availability Zone as your database — something not supported in a production environment, but often found in evaluation and early stages of development. For example, see Working in the Cloud in the Java SDK documentation. The documentation for each supported language links to client settings options, and to troubleshooting tools.

    Security Certificate

    Recent versions of the SDK (except the C SDK) — since API version 3.2 — ship with the client certificate for Capella, which enables an easy connection. This includes the versions recommended inChoose Your Language.

    Some platforms behave differently with where they store client certificates, and you may need a workaround with the following:

    • Python on Windows.

    • Node.js on Windows.

    • Node.js on macOS.

    The simplest fix is to append tls_verify=none to the connection string. This will skip the certificate verification step, but still use TLS to communicate between the client and Capella. This is for development environments only; it is not supported in production.

    With Node.js on macOS, for example, you can point at where the cert is on your system:

    CBPPLOGLEVEL=debug SSL_CERT_FILE=/opt/homebrew/etc/openssl@1.1/cert.pem node hello-world.js

    You can download a copy of the client certificate from the Capella UI, under Data Settings  Security Certificate.

    The Data API is another way of connecting to your Serverless Database Instance, giving a REST-like API to interact with your data.
