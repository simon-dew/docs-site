Request Prompt Action for Database Recovery

  • how-to
You can choose to authorize Couchbase Capella Support to automatically take remedial actions to a specific database, in case a failure is detected.

About Database Recovery Authorization

Use the Request prompt action for database recovery feature from the Settings tab of a deployed database. This authorization allows the Capella Support Team to take any necessary, time-sensitive actions to recover a failed or failing database as soon as possible.

If you enable this setting, Capella Support does not wait for your explicit permission to try and recover your database and any linked App Services. If you have not deployed App Services with your database, this authorization still applies to any future App Services deployments.

This setting is enabled by default when you create a new database.

Capella Support will only make changes to your database if you enable the database recovery authorization, and a delay in action might result in severe consequences to your database or App Services. You can Review Actions Taken By Capella Support at any time.

For more information about how to change the authorization on an existing database, see Enable Database Recovery Authorization for a Database.

Example Actions Capella Support Can Take On A Database

If you enable the database recovery authorization, Couchbase Capella support may do any of the following to try and recover your database and any linked App Services:

  • Increase storage size on disks.

  • Increase compute resources (CPU and RAM) on nodes.

  • Add a new node.

  • Add a new disk.

  • Create another database with the same topology, restored from the last known backup.

  • Create another database with increased storage and compute resources, restored from the last known backup.

Adding storage or compute resources to your database could result in additional costs to you.

What Actions Capella Support Cannot Take On A Database

If you enable the database recovery authorization, the Capella Support team still cannot view or change the data on your database.

Enable Database Recovery Authorization for a Database

You must have the Organization Owner, Project Owner, or Project Manager role to change the database recovery authorization for a database.

You must set this permission for each database in your organization. This setting also applies to any current or future App Services linked to the database.

To change the database recovery authorization for an existing database:

  1. In the Capella UI, on the Databases page, click the database you want to edit.

  2. Go to Settings  General.

  3. Under Request prompt action for database recovery, choose Yes.

  4. Click Save.

Review Actions Taken By Capella Support

All actions taken by Capella Support create a record in the Capella Activity Log.

For more information about how to view the Activity Log, see View Activity Logs.
