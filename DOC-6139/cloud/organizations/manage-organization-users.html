Mange Organization Users

    +
    Invite users to an organization and manage their access to projects and clusters.

    Managing Organization Users From the Couchbase Cloud UI

    All users in an organization can be viewed and managed from the Users tab in the main navigation. A summary of all users is displayed in table format.

    Users Summary

    The Users tab shows a summary of all users that have been added to the organization. The summary is displayed in table format, with sortable columns and a row for each user. A search field is provided at the top of the summary, which you can use to search for any term or value that is listed in the summary table. A checkbox is also provided for quickly filtering the user summary to only show pending users.

    The user summary displays the following information about each user:

    User

    The Full Name and Email address of the user. This column is sorted based on Full Name.

    Status

    The invitation status of the user: Verified or Pending.

    Type

    The type of user account: Couchbase Cloud user or Database user.

    Add/Invite Users to an Organization

    In order to add users to your organization, you must have Organization Admin privileges. You must also have the email address of the person you plan to invite.

    1. From the Users tab, click Add Users

      This opens the Add Users fly-out menu.

    2. Ensure that Add Couchbase Cloud User is checked.

      If you already know that the user you’re inviting will need access to certain cluster databases right away, you can add a database user for them by clicking the Add Database User checkbox. For more information about adding database users, refer to Configure Database Users.

    3. Enter the User Details.

      In the User Details section, configure the following:

      Full Name

      Enter the Full Name of the user that you’re inviting to your organization. This will be the name that other users will see in the Couchbase Cloud UI.

      If the user’s Email address is already associated with a Couchbase Cloud account, the Full Name that you specify here will be replaced with the Full Name that is already configured for that account.
      Email

      Enter the user’s Email address. This is the Email address that the invitation will be sent to. It will also be the Email address that the person will use to log into Couchbase Cloud.

      Organization Role

      Select an organization role for the user.

    4. (Optional) Add the user to projects.

      If you already know that the user you’re inviting will need access to certain projects, you can use the Add to Projects section to add the user directly to projects.

      To add the user to a project, use the Select projects drop-down menu to select the desired projects.

      In each of the projects that you selected, the user will be assigned the project role that is equivalent to their organization role. For example, if you assigned either the Organization Admin or Organization Edit role to the user, then they will be assigned the Project Edit role in each of the selected projects. Likewise, if you assigned the Organization View role to the user, then they will be assigned the Project View role in each of the selected projects.

    5. Once you’ve configured all of the required fields, click Add Users.

      Note that if you’re adding multiple users, leaving the Save and add another user checkbox selected will clear the fly-out menu after adding a user. This can be helpful when you need to add multiple users in a row.

    After you add a user, Couchbase Cloud sends that user an Email inviting them to join the organization. An invited user will display in the Users tab with a status of Pending until they accept their invitation.

    A pending user cannot access anything in the organization, including projects and clusters, until they accept the invitation and configure their account. Invitations expire after 24 hours.

    View Active and Pending Users in an Organization

    All users in an organization can be viewed from the Users tab in the main navigation. Each user is listed with a status of either Verified or Pending. All users that are listed with a status of Verified have accepted their invitation Emails and are allowed to access your organization. Users that are listed with a status of Pending have either not accepted their invitation, or their invitation has expired.

    You can use the Only show pending users checkbox to display only the users that have not yet accepted their invitations to your organization.

    If you have Organization Admin privileges, you will see all Database users in the entire organization.

    Non-Admin users will only see Database users if they are associated with a cluster that is contained within a project that they are a member of. This means that if a user doesn’t have Organization Admin privileges, and isn’t a member of any projects, then they won’t see any Database users in the Users tab.

    Resend or Cancel a User’s Invitation

    If a user didn’t receive their invitation before it expired, you have the option to resend the invitation. Alternatively, if you no longer wish to add the user to your organization, you have the option to cancel their invitation.

    1. From the Users tab, find the pending user that you wish to re-invite/cancel, and click on their row.

      Clicking on the user’s row opens the user’s fly-out menu.

      If you have trouble finding the desired pending user, click the Only show pending users checkbox to only display pending users. You can use the search field to narrow down the results further.

    2. To resend an invitation to the user, click Resend Invitation.

    3. If you no longer wish to add the user to your organization, you can cancel their invitation by clicking Remove User.

      When prompted to confirm the deletion of the invitation, click Confirm.

    Change a User’s Role in an Organization

    In order to change a user’s role in your organization, you must have Organization Admin privileges.

    An organization must have at least one user with the Organization Admin role at all times. If you are the only Organization Admin user, you cannot edit your own role until you grant the Organization Admin role to at least one other user.

    1. From the Users tab, find the user whose role you wish to change, and click on their row.

      Clicking on the user’s row opens the user’s fly-out menu.

    2. Click the Couchbase Cloud drop-down menu, and select the desired organization role for the user.

    3. Once you’ve selected the desired role, scroll to the bottom of the fly-out menu and click Save.

    Remove a User (or Yourself) from an Organization

    Use this procedure to remove either yourself, or another user, from an organization. To remove someone other than yourself from an organization, you must have Organization Admin privileges.

    An organization must have at least one user with the Organization Admin role at all times. If you are the only Organization Admin user, you cannot leave the organization until you grant the Organization Admin role to at least one other user.

    In addition, a project must have at least one member with the Project Edit role at all times. If the user that you are removing happens to be the only member with the Project Edit role in a particular project (even if that user is yourself), then you must first either grant the Project Edit role to another member of the project, or delete the project altogether.

    1. From the Users tab, find the user whom you wish to remove from the organization, and click on their row. (This can be yourself if you’re removing yourself from the organization.)

      Clicking on the user’s row opens the user’s fly-out menu.

    2. Scroll to the bottom of the user’s fly-out menu and click Remove User.

      When prompted to confirm the removal of the user, click Confirm.
