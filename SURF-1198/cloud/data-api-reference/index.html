Couchbase Capella Data API

      +

      Overview

      The Couchbase Capella Data API enables you to work with data resources, documents, and queries.

      Version information

      Version : 0.4.0

      Host information

      https://{database}.{host}

      You can find the Data API endpoint from the Data API section within the Couchbase Capella user interface. The Data API endpoint consists of the database name and the host where the database is located. The URL scheme must be https.

      Variable Description

      database

      The database name.

      Example: dapi

      host

      The host where the database is located.

      Example: couchbase.live

      This section describes the operations available with this REST API. The operations are grouped in the following categories.

      Description

      Fetch a list of documents, or retrieve documents in bulk, in the specified scope and collection.

      Path Parameters
      Name Description Schema

      required

      The name of the scope.

      String

      collectionName
      required

      The name of the collection.

      Query Parameters
      Name Description Schema

      optional

      A custom timeout for the operation.

      String (duration)

      pretty
      optional

      Whether the output should be formatted with indentations and newlines.

      Boolean

      meta
      optional

      Whether metadata should be included in the response.

      Boolean

      pagesize
      optional

      The number of documents to return in each page of results. Can be used with page to paginate the results.

      Integer

      page
      optional

      The page of results to return. Can be used with pagesize to paginate the results.

      Integer

      limit
      optional

      The number of documents to return. Identical to pagesize. Similar to the LIMIT clause in SQL++.

      Integer

      offset
      optional

      The number of documents to skip before returning results. Similar to the OFFSET clause in SQL++.

      Integer

      fields
      optional

      A list of field names, separated by comma. Only the specified fields are returned in the results.

      String

      where
      optional

      A condition that must be met for a document to be returned. Similar to the WHERE clause in SQL++.

      String

      orderBy
      optional

      The name of a field by which to sort the results. Similar to the ORDER BY clause in SQL++.

      String

      order
      optional

      Used with orderBy to specify the direction of the sort.

      String

      ids
      optional

      A list of document IDs, separated by comma. Only the specified documents are returned in the results.

      String

      exists
      optional

      Whether to check on a document’s existence. When true, the request merely checks that matching documents exist. When false, the request returns all matching documents.

      Create Document

      Description

      Creates or updates a document, identified by its unique ID, in the specified scope and collection.

      Parameters
      Path Parameters
      Name Description Schema

      scopeName
      required

      The name of the scope.

      String

      collectionName
      required

      The name of the collection.

      String

      docId
      required

      The unique ID of the document.

      String
      Body Parameter
      Name Description Schema

      Body
      required

      The document content.

      Example: {"id":10,"name":"40-Mile Air","iata":"Q5","icao":"MLA","country":"United States","accessCount":50}

      AnyType
      Query Parameters
      Name Description Schema

      timeout
      optional

      A custom timeout for the operation.

      String (duration)

      expiry
      optional

      The expiration time of the document.

      String

      optional

      Specifies whether documents should keep their current expiration setting if modified by the request.

      Not supported for SQL++ statements in a transaction.

      durability
      optional

      meta
      optional

      Whether metadata should be included in the response.

      Boolean
