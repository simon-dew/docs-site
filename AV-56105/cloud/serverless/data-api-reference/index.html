Couchbase Capella Data API

  • reference
    +

    Overview

    The Couchbase Capella Data API enables you to work with data resources, documents, and queries.

    Version information

    Version : 0.4.0

    Host information

    https://{database}.{host}

    You can find the Data API endpoint from the Data API section within the Couchbase Capella user interface. The Data API endpoint consists of the database name and the host where the database is located. The URL scheme must be https.

    Variable Description

    database

    The database name.

    Example: dapi

    host

    The host where the database is located.

    Example: couchbase.live

    Resources

    This section describes the operations available with this REST API. The operations are grouped in the following categories.

    Metadata

    Delete Bulk Documents

    DELETE /v1/scopes/{scopeName}/collections/{collectionName}/docs
    Description

    Deletes documents in bulk in the specified scope and collection.

    Parameters
    Path Parameters
    Name Description Schema

    scopeName
    required

    The name of the scope.

    String

    collectionName
    required

    The name of the collection.

    String
    Body Parameter
    Name Description Schema

    Body
    optional

    An array of strings, each of which is the ID of a document to delete.

    Example: ["x_y_z2","abc2"]

    String List
    Query Parameters
    Name Description Schema

    pretty
    optional

    Whether the output should be formatted with indentations and newlines.

    Boolean

    meta
    optional

    Whether metadata and errors should be included in the response.

    Boolean
    Produces

    • application/json

    Consumes

    • application/json

    Responses
    HTTP Code Description Schema

    200

    Success. Returns an object containing a nested array of deleted document IDs.

    Bulk Modify Response

    401

    Unauthorized.

    Error
    Security
    Type Name

    http (basic)

    Basic
    Example HTTP Request

    Delete two documents in the airline collection and format the output.

    Curl request
    curl -X DELETE \
     -H 'Content-Type:application/json' \
     -u "${api_key}:${secret_key}" \
     -d '["x_y_z2","abc2"]' \
"${endpoint}/v1/scopes/samples/collections/airline/docs?pretty=true"

    • Replace ${api_key} and ${secret_key} with your API key and Secret key. See Security.

    • Replace ${endpoint} with the URL of your Data API endpoint. See Host information.

    Example HTTP Response
    Response 200
    {
  "docs" : [ {
    "id" : "x_y_z2"
  }, {
    "id" : "abc2"
  } ]
}

    List Documents or Get Bulk Documents

    GET /v1/scopes/{scopeName}/collections/{collectionName}/docs
    Description

    Fetch a list of documents, or retrieve documents in bulk, in the specified scope and collection.

    Parameters
    Path Parameters
    Name Description Schema

    scopeName
    required

    The name of the scope.

    String

    collectionName
    required

    The name of the collection.

    String
    Query Parameters
    Name Description Schema

    timeout
    optional

    A custom timeout for the operation.

    String (duration)

    pretty
    optional

    Whether the output should be formatted with indentations and newlines.

    Boolean

    meta
    optional

    Whether metadata should be included in the response.

    Boolean

    pagesize
    optional

    The number of documents to return in each page of results. Can be used with page to paginate the results.

    Integer

    page
    optional

    The page of results to return. Can be used with pagesize to paginate the results.

    Integer

    limit
    optional

    The number of documents to return. Identical to pagesize. Similar to the LIMIT clause in SQL++.

    Integer

    offset
    optional

    The number of documents to skip before returning results. Similar to the OFFSET clause in SQL++.

    Integer

    fields
    optional

    A list of field names, separated by comma. Only the specified fields are returned in the results.

    String

    where
    optional

    A condition that must be met for a document to be returned. Similar to the WHERE clause in SQL++.

    String

    orderBy
    optional

    The name of a field by which to sort the results. Similar to the ORDER BY clause in SQL++.

    String

    order
    optional

    Used with orderBy to specify the direction of the sort.

    String

    ids
    optional

    A list of document IDs, separated by comma. Only the specified documents are returned in the results.

    String

    exists
    optional

    Whether to check on a document’s existence. When true, the request merely checks that matching documents exist. When false, the request returns all matching documents.

    Boolean
    Produces

    • application/json

    Responses
    HTTP Code Description Schema

    200

    Success. Returns an object containing documents and metadata.

    Bulk Lookup Response

    401

    Unauthorized.

    Error
    Security
    Type Name

    http (basic)

    Basic
    Example HTTP Request

    Get two documents from the airline collection, including metadata, and format the output.

    Curl request
    curl -X GET \
     -H 'Content-Type:application/json' \
     -u "${api_key}:${secret_key}" \
"${endpoint}/v1/scopes/samples/collections/airline/docs?ids=x_y_z2,abc2&pretty=true&meta=true"

    • Replace ${api_key} and ${secret_key} with your API key and Secret key. See Security.

    • Replace ${endpoint} with the URL of your Data API endpoint. See Host information.

        }
      }
    },
    {
      "id": "abc2",
      "doc": {
        "country": "India",
        "name": "abc2"
      },
      "meta": {
        "api": {
          "consistencyToken": "1eyJjYXMiOjE2Nzg0Njk2NDMyNTg0Mjk0NDAsIm11dGF0aW9uU3RhdGUiOm51bGx9Cg=="

    Create Bulk Documents

    POST /v1/scopes/{scopeName}/collections/{collectionName}/docs
    Description

    Creates documents in bulk in the specified scope and collection.

    Parameters
    Path Parameters
    Name Description Schema

    scopeName
    required

    The name of the scope.

    String

    collectionName
    required

    The name of the collection.

    String
    Body Parameter
    Name Description Schema

    Body
    optional

    An array of objects, each of which contains the ID and value of a document to create.

    Bulk Modify Request List
    Query Parameters
    Name Description Schema

    upsert
    optional

    Whether to update existing documents.

    When true, if a document already exists, the document is overwritten. When false, if a document already exists, the operation fails.

    Boolean

    pretty
    optional

    Whether the output should be formatted with indentations and newlines.

    Boolean

    meta
    optional

    Whether metadata and errors should be included in the response.

    Boolean
    Produces

    • application/json

    Consumes

    • application/json

    Responses
    HTTP Code Description Schema

    200

    Success. Returns an object containing a nested array of created document IDs.

    Bulk Modify Response

    401

    Unauthorized.

    Error
    Security
    Type Name

    http (basic)

    Basic
    Example HTTP Request

    Create two documents in the airline collection and format the output.

    Curl request
    curl -X POST \
     -H 'Content-Type:application/json' \
     -u "${api_key}:${secret_key}" \
     -d '[ {"id": "x_y_z2", "value": {"name": "xyz2", "country": "US"}},
           {"id": "abc2", "value": {"name": "abc2", "country": "India"}} ]' \
"${endpoint}/v1/scopes/samples/collections/airline/docs?pretty=true"

    • Replace ${api_key} and ${secret_key} with your API key and Secret key. See Security.

    • Replace ${endpoint} with the URL of your Data API endpoint. See Host information.

    Example HTTP Response
    Response 200
    {
  "docs" : [ {
    "id" : "x_y_z2"
  }, {
    "id" : "abc2"
  } ]
}

    • upsert — Unconditionally set the value at the path.
