Couchbase Capella Management API

  • reference
    +

    Overview

    The Couchbase Capella Management API provides a set of OpenAPIs for creating and managing Capella instances. It enables users to perform operations such as creating new Capella instances, managing their configurations, and interacting with the Capella services. This API documentation specifies the endpoints, request/response formats, and authentication requirements for seamless integration with Couchbase Capella.

    Version information

    Version : v4.0

    Host information

    https://cloudapi.cloud.couchbase.com

    Create AllowedCIDR

    POST /v4/organizations/{organizationId}/projects/{projectId}/clusters/{clusterId}/allowedcidrs
    Description

    Adds a trusted CIDR to a cluster’s list of allowed CIDRs.

    In order to access this endpoint, the provided API key must have at least one of the roles referenced below: - Organization Owner - Project Owner - Project Manager

    To learn more, see our docs [here](https://docs.couchbase.com/cloud/organizations/organization-projects-overview.html).

    Note that updating this resource is not supported; you must delete and recreate allowed CIDRs instead. As a result, ETags are also not supported for this resource.

    Couchbase Server provides scopes and collections; allowing documents to be categorized and organized, within a bucket. A scope is a mechanism for the grouping of multiple collections. Up to 1000 scopes can be created per cluster. Collection-names must be unique within their scope. Collections might be assigned to different scopes according to content-type, or to deployment-phase (ie, test versus production). Applications can be assigned per-scope access-rights; allowing each application to access only those scopes it requires.
