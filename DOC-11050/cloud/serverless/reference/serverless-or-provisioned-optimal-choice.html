Choosing Capella Serverless or Provisioned

    Capella Provisioned and Serverless databases offer different optimizations of price and features.

    Use the Capella Serverless Private Preview to get an early look at Couchbase’s Serverless product: a Database-as-a-service (dBaaS). NOTE: Capella Serverless is not supported for production deployments, and not recommended for heavy read or write loads.

    Use a Serverless database to reduce your administration load, such as the cost of managing infrastructure.

    Serverless Private Preview

    A Provisioned Cluster gives you more control, and the ability to cope with a higher workload; you can also choose clusters in each of the major cloud providers. With this comes control of multi-region deployments.

    In the Serverless Private Preview, most aspects of your deployment are managed by Couchbase. If you want to trial Couchbase, it’s recommended to use a Serverless database.

    Features will be continuously added before the full release of Serverless. before making your choice (via the tab in the Deployment screen), read the section below, with its feature-comparison table.

    Serverless vs Provisioned Features

    Some features are not available in the preview version of Serverless, but may be added later. Some features will not be available for operational or security reasons. The following table lists key differences between the current preview of Serverless and Capella Provisioned databases:

    Table 1. Capella Feature Availability in Serverless & Provisioned

    Feature

    Serverless

    Provisioned

    Cloud Providers

    AWS

    AWS, Azure, and GCP.

    Database

    A Serverless database is able to hold up to 1000 Scopes and 1000 Collections.

    A Provisioned database can contain up to 30 buckets, each with up to 1000 Scopes and 1000 Collections.

    Index Management

    Indexes are fully managed by Couchbase. One replica is the default.

    You can create replicas of indexes to enhance availability.

    Search Index Aliases

    Not available for Serverless instances.

    Create an index alias for one or more Search indexes. Run a search against an alias to get the results from more than one index at a time.

    SQL++ CURL Function

    Not available for Serverless instances.

    The CURL() function implements a subset of curl functionality and enables SQL++ queries to interact and integrate with external JSON data sources available over HTTP/REST.

    System Catalog Access

    A user can’t access:

    • Monitoring catalogs

      1. completed_request

      2. prepared

      3. Active_requests

      4. Dictionary

      5. Dictionary_cache

      6. Functions_cache

      7. Task_cache

      8. transactions

    • Security catalogs

      1. Nodes

      2. applicable_roles

    • Other

      1. Vitals

    A user can access monitoring catalogs with the Bucket-level admin role.

    Cross Data Center Replication (XDCR)

    Not available for Serverless instances.

    XDCR allows data to be replicated across databases that are potentially located in different clouds and different data centers.

    Analytics Service

    Not available for Serverless instances.

    Yes

    Eventing Service

    Not available for Serverless instances.

    Yes

    JavaScript Query Functions

    Not available for Serverless instances.

    Yes, see the 7.2@JavaScript Functions Guide.

    Migration

    Currently no managed path to/from Serverless — use command line tools.

    Currently no managed path to/from Serverless — use command line tools.

    Serverless Limitations

    • A Serverless instance consists of a single Couchbase bucket. You can create up to 1000 Scopes and 1000 Collections.

    • The Capella Serverless Private Preview will limit the size of some database interactions. For more information about these limits, see Limits and Throttling.
